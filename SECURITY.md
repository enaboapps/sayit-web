# Security Policy

## Supported Versions

Security fixes target the latest `main` branch and the latest public release.

## Reporting A Vulnerability

Do not open a public GitHub issue for exploitable security vulnerabilities.

Report vulnerabilities privately by using GitHub private vulnerability reporting if it is enabled for this repository, or by contacting the maintainers through the security contact listed on the repository.

Please include:

- The affected route, component, API endpoint, or workflow
- Reproduction steps
- The expected and actual behavior
- The security impact
- Any suggested mitigation, if known

## Secrets

Secrets accidentally committed by contributors must be treated as compromised. Remove the secret from code, rotate it with the provider, and document the remediation in the relevant issue or private security report.
